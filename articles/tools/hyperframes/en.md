# HyperFrames

`HyperFrames` is a video toolchain for agents.

It is not a regular video generation model, and it is not a video editor. Think of it as writing a video with HTML/CSS/JavaScript, then previewing and rendering it to `MP4` through a CLI.

The Codex App plugin is only one entry point. You can also use HyperFrames from the CLI with Codex or Claude Code.

If you only want to enable the HyperFrames plugin inside Codex App, start with [Runtime / Codex App Plugin Setup](/docs/runtime/codex-plugins). That page covers grey plugin entries, missing `/plugins` entries, and the `hyperframes@openai-curated` config name.

This page focuses on HyperFrames itself: when to use it, and how to initialize, preview, and render a video project.

<h2 id="when-to-use">When to Use It</h2>

Use it for:

- product launch videos
- course intros or section transitions
- social motion graphics
- turning web-style visuals into MP4
- reusable video compositions generated by an agent

If you only need a static image, start with [SorryCode Image2](/docs/skills/sorrycode-image2). If you need a presentation deck, start with [Magazine Web PPT](/docs/skills/magazine-web-ppt) or [Open Slide](/docs/tools/open-slide).

<h2 id="how-it-works">How It Works</h2>

The usual flow:

```text
initialize video project -> ask the agent to write the composition -> preview locally -> render MP4
```

Common commands:

```bash
npx hyperframes init my-video
cd my-video
codex
```

After the agent writes the video, preview and export it:

```bash
npx hyperframes preview
npx hyperframes render --output final.mp4
```

<h2 id="first-prompt">First Prompt</h2>

Inside the project, say:

```text
Use HyperFrames to make a 20-second product launch video. First give me the script, storyboard, visual direction, and pacing. Do not render yet.
```

If you already have a webpage or product description:

```text
Use HyperFrames to turn this product description into a 15-second social video. Design 4 shots first, with copy, visuals, and motion for each shot.
```

<h2 id="before-start">Before You Start</h2>

Check your environment first:

```bash
node -v
npx hyperframes doctor
```

If Node.js is missing, start with [Environment / Node.js](/docs/environment/nodejs).

<h2 id="how-to-choose">How to Choose</h2>

| Need | Start with |
| --- | --- |
| Static covers, posters, or illustrations | [SorryCode Image2](/docs/skills/sorrycode-image2) |
| Web-style or magazine-style presentation decks | [Magazine Web PPT](/docs/skills/magazine-web-ppt) |
| Code-based React decks | [Open Slide](/docs/tools/open-slide) |
| App prototypes, motion, infographics, and critique | [Huashu Design](/docs/skills/huashu-design) |
| MP4 short videos and web-style motion graphics | `HyperFrames` |

<h2 id="not-for">When Not to Use It</h2>

Do not start here for:

- long-form video editing
- complex live-action post-production
- heavy audio tracks, subtitles, and editing timelines
- one-prompt live-action video generation

HyperFrames is better for controllable web-style short videos and motion graphics.
