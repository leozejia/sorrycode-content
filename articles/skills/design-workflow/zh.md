# 设计类 Skills 怎么用

SorryCode 提供的是方法和渔具，不生产鱼。

设计、PPT、图片、写作、一页纸和封面这类产物，都带有很强的主观判断。1000 个人会有 1000 种偏好：有人嫌字小，有人嫌太空，有人喜欢杂志感，有人只要商业克制。

所以设计类 skill 不是“一句话成品机”。它的价值，是让 agent 给你一个结构化、可继续修改的强起点。最终要不要改、往哪里改、改到什么程度，需要你用自己对主题、受众、场景和审美的掌握来判断。

<h2 id="mindset">先换一个心智</h2>

不要把设计类 skill 当成外包设计师，也不要把 agent 当成最终审美裁判。

更准确的协作方式是：

1. 你给目标、受众、场景、素材和约束
2. skill 让 agent 生成一个初稿或路线
3. 你判断哪里不对
4. agent 按你的反馈局部调整
5. 高频出现的偏好，沉淀成自己的 `DESIGN.md`、references 或 skill

一句话生成可以用来开始，不应该用来结束。

<h2 id="three-feedback-types">把反馈拆成三类</h2>

当你觉得“不好看”“不对味”“字太小”，先判断它属于哪一类。

| 类型 | 例子 | 怎么处理 |
| --- | --- | --- |
| 质量问题 | 字太小、对比不够、图片糊、排版溢出、导出失败 | 直接让 agent 修，这是下限问题 |
| 场景问题 | 不适合投影、不适合老板、不适合销售、不符合品牌 | 补充受众、场合和限制，再让 agent 调整 |
| 偏好问题 | 我想更极简、更冲击、更东方、更商业、更像某个参考 | 说清方向，反复出现就沉淀成自己的规则 |

这一步很重要。否则你只说“风格不好”，agent 很难知道是质量失败、场景错配，还是你的个人偏好。

<h2 id="how-to-iterate">怎么让 agent 快速改对</h2>

不要每次都重做整套。先缩小范围，再说清楚判断标准。

```text
先不要重做整套。只调整第 3-6 页：
1. 字号提高到投影可读
2. 保留现有结构
3. 减少装饰元素
4. 风格从杂志感改成更商业、更克制
5. 改完后列出你调整了哪些页面
```

如果你不确定问题在哪里，可以先让 agent 诊断。

```text
请先不要修改。帮我审查这套 PPT，把问题分成三类：
1. 质量问题
2. 场景问题
3. 个人偏好可能导致的分歧

每类只列最重要的 3 个问题，然后问我先改哪一类。
```

<h2 id="choose-path">先选路线，再选工具</h2>

设计类任务不要一上来就问“哪个 skill 最强”。先问你要交付什么。

| 你要交付 | 优先看 |
| --- | --- |
| 封面、海报、插图、产品视觉 | [SorryCode Image2](/docs/skills/sorrycode-image2) |
| 一页纸、简历、报告、白皮书、作品集 | [Kami](/docs/skills/kami) |
| 有明确设计风格的网页演示 | [藏师傅的 PPT Skill](/docs/skills/magazine-web-ppt) |
| App 原型、动效、信息图、设计变体或专家评审 | [Huashu Design](/docs/skills/huashu-design) |
| 修改已有 PowerPoint 文件 | [PPTX](/docs/skills/pptx) |
| 代码式、可维护、可评论的 deck | [工具 / Open Slide](/docs/tools/open-slide) |

如果只是第一次尝试做一套分享稿，可以走 [新手村 / 做一套分享 PPT](/docs/village/share-ppt)。那里会给默认路径，不要求你先理解所有工具。

<h2 id="make-it-yours">把自己的偏好沉淀下来</h2>

当你发现自己反复说这些话，就不要每次临场补充：

- 字号必须适合投影
- 不要大面积渐变
- 不要太像营销海报
- 中文标题要更有呼吸感
- 页面要更像咨询公司汇报
- 所有封面都要保留某个品牌色

这些不是一次性 prompt，应该变成长期规则。

最轻量的做法是写一个 `DESIGN.md`。它可以记录品牌色、字体、参考案例、禁用风格、交付格式和验收标准。下一步看：[Agent 基建 / DESIGN.md](/docs/agent-infra/design-md)。

如果这套流程已经反复使用，就把它做成自己的 skill。下一次你不需要解释全部背景，只要点名自己的 skill，让 agent 按你的方法开始。

<h2 id="first-prompt">第一句话可以怎么说</h2>

如果你还不知道该用哪个 skill：

```text
我想做一个可以交付给别人的设计类产物。请先判断我应该用 SorryCode Image2、Kami、藏师傅的 PPT Skill、PPTX、Huashu Design 还是 Open Slide。不要直接生成，先问我目标、受众、场景、素材和风格偏好。
```

如果你已经有初稿：

```text
请先审查这份初稿，不要直接重做。把问题分成质量问题、场景问题、个人偏好三类，然后给我一个最小修改方案。
```

如果你想沉淀自己的规则：

```text
下面是我连续几次对设计稿的修改意见。请帮我整理成一份 DESIGN.md，区分长期规则、可选偏好和这次任务专属要求。
```
