# 设计类 Skills 怎么用

设计类 skills 适合把你从空白页带到一个可看的初稿，并让后续修改更有方向。

PPT、图片、写作、一页纸、封面和产品视觉，很少靠一句话直接定稿。它们通常要根据受众、场景、品牌、阅读方式和个人偏好继续调整。这不是 skill 失效，而是设计类工作的正常过程。

你越能说清楚哪里不对，agent 越容易帮你改到位。

<h2 id="what-it-solves">它先解决什么问题</h2>

设计类任务最难的部分，通常不是“会不会画”或“会不会排版”，而是从一团模糊需求进入可讨论状态。

skill 的价值在这里：

1. 帮你启动，不用从空白页开始
2. 帮 agent 记住稳定流程，比如先问问题、再给路线、最后保存文件
3. 帮你得到一个可以指出问题的版本
4. 帮你把修改控制在局部，而不是每次重做
5. 帮你把反复出现的偏好沉淀成长期规则

所以第一次生成的结果，不一定是终点。更常见的情况是：它先给你一个可以继续判断和修改的版本。

<h2 id="why-iterate">为什么通常需要迭代</h2>

同一份内容，放在不同场景里，会有不同答案。

| 场景变化 | 设计会怎么变 |
| --- | --- |
| 投影演讲 | 字号、对比度和页面节奏更重要 |
| 发给客户阅读 | 结构、可信度和信息完整度更重要 |
| 社媒封面 | 第一眼识别和标题清晰度更重要 |
| 内部汇报 | 克制、准确和可快速扫描更重要 |
| 个人作品 | 风格记忆点和表达气质更重要 |

如果这些背景没有说清楚，agent 只能根据默认判断先做一个版本。后续迭代的作用，就是把结果从“默认可用”推向“适合你这次场景”。

<h2 id="three-feedback-types">把反馈拆成三类</h2>

当你觉得“不好看”“不对味”“字太小”，先把问题拆开。

| 类型 | 例子 | 怎么反馈 |
| --- | --- | --- |
| 质量问题 | 字太小、对比不够、图片糊、排版溢出、导出失败 | 直接指出问题，让 agent 修到可用 |
| 场景问题 | 不适合投影、不适合客户、不适合销售、不符合品牌 | 补充受众、场合、品牌和交付限制 |
| 偏好问题 | 想更极简、更克制、更有冲击力、更接近某个参考 | 给方向或参考，必要时沉淀成长期规则 |

这比只说“风格不好”更有效。agent 不需要猜你真正介意的是什么，它可以直接根据问题类型调整。

<h2 id="how-to-iterate">怎么让 agent 改得更快</h2>

先缩小范围，再说清楚判断标准。很多时候不需要重做整套。

```text
先不要重做整套。只调整第 3-6 页：
1. 字号提高到投影可读
2. 保留现有结构
3. 减少装饰元素
4. 风格从杂志感改成更商业、更克制
5. 改完后列出你调整了哪些页面
```

如果你还不知道问题在哪里，可以先让 agent 帮你审查。

```text
请先不要修改。帮我审查这套 PPT，把问题分成三类：
1. 质量问题
2. 场景问题
3. 偏好问题

每类只列最重要的 3 个问题，然后问我先改哪一类。
```

<h2 id="choose-path">先选路线，再选工具</h2>

设计类任务可以先问“我要交付什么”，再决定用哪个 skill 或工具。

| 你要交付 | 优先看 |
| --- | --- |
| 封面、海报、插图、产品视觉 | [SorryCode Image2](/docs/skills/sorrycode-image2) |
| 一页纸、简历、报告、白皮书、作品集 | [Kami](/docs/skills/kami) |
| 有明确设计风格的网页演示 | [藏师傅的 PPT Skill](/docs/skills/magazine-web-ppt) |
| 修改已有 PowerPoint 文件 | [PPTX](/docs/skills/pptx) |
| 代码式、可维护、可评论的 deck | [工具 / Open Slide](/docs/tools/open-slide) |

如果只是第一次尝试做一套分享稿，可以走 [新手村 / 做一套分享 PPT](/docs/skills/magazine-web-ppt)。那里会给默认路径，不要求你先比较所有工具。

<h2 id="make-it-yours">把稳定偏好写下来</h2>

如果你反复提出类似修改，就可以把它们写成长期规则：

- 字号要适合投影
- 不要大面积渐变
- 不要太像营销海报
- 中文标题要有呼吸感
- 页面要更像咨询公司汇报
- 封面要保留某个品牌色

最轻量的做法是写一个 `DESIGN.md`。它可以记录品牌色、字体、参考案例、禁用风格、交付格式和验收标准。下一步看：[Agent 基建 / DESIGN.md](/docs/agent-memory/remember-design)。

如果这套流程已经反复使用，再考虑做成自己的 skill。下一次你只要点名这个 skill，agent 就能按你的方法开始。

<h2 id="first-prompt">第一句话可以怎么说</h2>

如果你还不知道该用哪个 skill：

```text
我想做一个可以交付给别人的设计类产物。请先判断我应该用 SorryCode Image2、Kami、藏师傅的 PPT Skill、PPTX、Open Slide 还是 HyperFrames。不要直接生成，先问我目标、受众、场景、素材和风格偏好。
```

如果你已经有初稿：

```text
请先审查这份初稿，不要直接重做。把问题分成质量问题、场景问题、偏好问题三类，然后给我一个最小修改方案。
```

如果你想沉淀自己的规则：

```text
下面是我连续几次对设计稿的修改意见。请帮我整理成一份 DESIGN.md，区分长期规则、可选偏好和这次任务专属要求。
```
