# Who SorryCode Is For

SorryCode is not another chat web app.

It is an AI tool balance and API entry point. You create API keys, connect tools such as `Codex`, `Claude Code`, Gemini paths, image APIs, and Skills, then use one balance across them.

If you are not sure which AI product to subscribe to, try the workflow through SorryCode first.

## The Short Version

If you spend roughly `$1-$100/month` on AI tools and do not want to lock yourself into one subscription, SorryCode is worth trying.

This is not a minimum spend.

You can spend `$1` just to test a workflow. You can also use it regularly at much higher volume. The point is not that you must spend a lot. The point is that one balance can reach multiple tools.

## Three Common User Types

### 1. Trial users: $1-$10/month

You want to try:

- whether Codex can read a project
- what Claude Code feels like
- what Gemini or image models can do
- whether Skills help with documents, slides, images, or workflows

You do not need to subscribe to a full month before learning the answer.

Add a small balance, run the workflow, then decide.

### 2. Light-to-medium users: $10-$30/month

You already use AI as a daily tool for:

- writing and research
- office documents
- reading code projects
- image or cover generation
- small agent tasks

But you do not want to subscribe to every platform separately.

SorryCode gives you one balance across multiple entries.

### 3. Heavy but not enterprise users: $30-$100/month

You often use agents for projects, file edits, and long tasks, but you are not ready to buy every Pro / Max plan.

Subscriptions can become awkward:

- ChatGPT only, but you also want Claude Code
- Claude only, but you also want Codex and GPT
- multiple subscriptions, but uneven usage
- fixed monthly fees, but your workload changes

SorryCode fits people who use AI often but do not want one product to define the whole workflow.

## When an Official Subscription Is Better

If you use one official product every day and reliably consume its subscription allowance, the official subscription may be better.

For example:

- you only use the ChatGPT web app
- you only use the Claude web app
- you know one product covers your needs
- you do not need Codex, Claude Code, Skills, image APIs, or multi-model switching

SorryCode is not a replacement for every subscription.

It is better for exploration, mixed tool use, uneven usage, and users who want to avoid subscribing to multiple products too early.

## What SorryCode Gives You

### One balance

You do not need a separate monthly subscription for every platform you want to try.

### Multiple API keys

Create separate `sk-...` keys by use.

For example, one for Codex, one for Claude Code, and one for an image skill. They consume the same balance, but usage records, groups, limits, and troubleshooting stay clearer.

### Multiple entries

Choose by task:

- GPT path: start with [Runtime / Codex](/docs/runtime/codex)
- Claude path: start with [Runtime / Claude Code](/docs/runtime/claude-code)
- Images: read [Skills / SorryCode Image2](/docs/skills/sorrycode-image2)
- Office files: read [Skills / Office Docs](/docs/skills/office-docs)

### Easier experimentation

Start small, test the workflow, then decide what is worth using more.

## You Do Not Need to Understand APIs First

If the word API feels unfamiliar, use this first:

```text
API Key = the key that lets a tool use your SorryCode balance
```

You do not need to write API requests on day one. Create the key you need, then install Codex or Claude Code based on your goal.
