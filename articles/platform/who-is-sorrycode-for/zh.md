# SorryCode 适合谁

SorryCode 不是另一个聊天网页。

它更像一个 AI 工具额度入口：你创建 API Key，把 `Codex`、`Claude Code`、图片生成和 Skills 接进来，用一份余额体验多个工具。

如果你还不确定自己该订阅哪个 AI 产品，先用 SorryCode 试一圈，通常比一上来买多个月费更轻。

## 一句话判断

如果你每月 AI 工具花费大约在 `1-700 元`，并且不想被单一订阅绑死，SorryCode 就值得先试。

这不是最低消费门槛。

你花 `1 元` 尝鲜也可以；你每月稳定用到几百元也可以。关键不是你必须用很多，而是你想用一份余额跨多个工具。

## 三类典型用户

### 1. 尝鲜用户：1-50 元/月

你只是想试试：

- Codex 能不能帮你读项目
- Claude Code 是什么体验
- 在 Codex App 中生成图片是什么体验
- Skills 能不能帮你做文档、PPT、图片

这种情况下，不一定要先订阅一个完整月。

先充一点余额，把流程跑通，再决定要不要长期用。

### 2. 轻中度用户：50-200 元/月

你已经开始把 AI 当日常工具用：

- 写作和整理资料
- 处理办公文件
- 读代码项目
- 生成图片或封面
- 让 agent 帮你完成小任务

但你还不想每个平台都订阅。

SorryCode 的价值是：一份余额，可以接多个入口。

### 3. 高频用户：200-700 元/月

你经常用 agent 做项目、改文件、跑任务，但还没有高频到每个平台都买 Pro / Max。

这时订阅制常常很尴尬：

- 只买 ChatGPT，又想用 Claude Code
- 只买 Claude，又想试 Codex 和 GPT
- 买多个订阅，又不一定每天都吃满额度
- 用量有高峰和低谷，月费却是固定的

SorryCode 更适合这种“高频但不想被单一产品锁死”的用法。

## 什么时候更适合官方订阅

如果你每天都稳定使用同一个官方产品，而且能吃满它的订阅额度，官方订阅可能更适合。

比如：

- 每天都只用 ChatGPT 网页
- 每天都只用 Claude 网页
- 非常明确自己只需要一个产品
- 不需要 Codex、Claude Code、Skills、图片接口或多模型切换

SorryCode 不是要替代所有订阅。

它更适合还在探索、多工具混用、用量波动、或者不想一次绑定多个订阅的人。

## SorryCode 帮你解决什么

### 一份余额

你不需要分别给每个平台都买一份月费。

### 多把 API Key

你可以按用途创建多个 `sk-...`。

比如 Codex 一把，Claude Code 一把，图片 Skill 一把。它们消耗同一份余额，但使用记录、分组、限额和排障会分得更清楚。

### 多个入口

你可以按任务选择入口：

- GPT 路径：优先看 [Runtime / Codex](/docs/runtime/codex)
- Claude 路径：优先看 [Runtime / Claude Code](/docs/runtime/claude-code)
- OpenAI 图片：先看 [模型与工作台 / GPT Image 2](/docs/runtime/gpt-image-2)
- Grok 图片和视频：看 [Grok 图片生成](/docs/runtime/grok-image) 与
  [Grok 视频生成](/docs/runtime/grok-video)
- 办公文档：看 [Skills / 办公文档](/docs/skills/office-docs)

### 更适合试错

你可以先小额尝试，再决定哪个工具真正适合你。

## 你不需要先懂 API

很多人看到 API 就紧张。

新手可以先这样理解：

```text
API Key = 让工具使用你 SorryCode 余额的钥匙
```

你不需要第一天就会写代码请求。先按目标创建 Key，再安装 Codex 或 Claude Code。
