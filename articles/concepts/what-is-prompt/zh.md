# 跟 AI 说话，不是念咒

网上有一种内容特别流行：

```text
"分享 10 个神级 Prompt！"
"这个 Prompt 让 AI 效率提升 10 倍！"
"收藏！最全 Prompt 模板大全！"
```

看起来好像跟 AI 说话需要学一门新语言，需要背诵魔法咒语。

不需要。

<h2 id="what-it-is">Prompt 就是工作指令</h2>

回想一下你给同事派活的时候怎么说的。

你不会说"做一个好的报表"。你会说：

```text
"帮我把这个月的销售数据整理一下。
按产品线分类，算环比增长，异常数据标红。
明天下班前放我桌上。"
```

三句话，四个关键信息：

```text
1. 做什么：整理销售数据，按产品线分类，算环比增长
2. 怎么做：异常数据标红
3. 什么算做完：明天下班前放桌上
4. 背景信息：这个月的，销售数据（他知道在哪里）
```

给 AI 的指令，完全一样的结构。

```text
好 Prompt = 任务 + 边界 + 验收标准 + 背景
```

网上那些"神级 Prompt"之所以看起来很长很复杂，不是因为 AI 需要复杂的话。是因为写的人把背景信息、边界条件、验收标准都写清楚了。

你也可以。

<h2 id="four-parts">四个要素</h2>

把任何一个任务拆成四个部分，你的 Prompt 就不会差：

**1. 任务描述——我要什么**

越具体越好。不要说"帮我写个报告"，说"帮我写一份 500 字以内的季度工作总结，面向部门总监汇报"。

**2. 边界——我不要什么**

告诉 AI 什么不能做、什么不能写、什么风格不行。比告诉它"要做什么"更管用的是告诉它"别做什么"。

比如："不要用技术术语，这是给销售看的。不要超过一页纸。不要提及竞品名字。"

**3. 验收标准——什么算做完**

AI 不知道什么时候它算"完成了"。你告诉它：格式是什么、长度是多少、要包含哪几个部分。

比如："输出一份 Markdown 文件，包含三个部分：业绩概述、问题分析、下季度计划。"

**4. 背景信息——你需要知道什么**

AI 没有常识，不知道你的上下文。你给它什么信息，它就只能基于这些信息工作。

比如："我们上季度目标 500 万，实际完成 420 万。主要原因是 3 月大客户延期签约。团队一共 8 个人。"

```text
这四个要素不必每次都写全。
但如果你觉得 AI 的回复不在点子上，
检查一下——是不是这四个里漏了什么。
```

<h2 id="examples">两个例子，一眼看懂</h2>

**模糊的指令：**

```text
帮我写一份产品介绍。
```

AI 不知道你的产品是什么、给谁看、多长、什么风格。它会猜。猜的结果大概率不是你想要的。

**清晰的指令：**

```text
帮我写一份产品介绍。产品是工业用的不锈钢阀门，面向工厂采购经理。300 字以内，不要用技术参数堆砌，要讲清楚给客户省了什么。不要提具体价格。输出纯文字，不要表格。
```

区别就是：上面那个你要来回改五遍。下面这个大概率一遍过。

<h2 id="iterate">好的 Prompt 是改出来的，不是写出来的</h2>

这是最重要的心态。

```text
不要指望第一次就把 Prompt 写完美。
第一版 Prompt = 启动对话，不是终极指令。
```

你跟新员工交代任务的时候，第一遍说完，他可能理解偏了。你说"不是这个意思，是这样..."，他调整一下。一两轮之后，你们对齐了。

AI 也一样。

第一遍，你说个大概。它给个初稿。

第二遍，你说"这里不对，应该改成..."。它调整。

第三遍，你说"好，现在整体风格再改一下..."。它再调整。

三遍之后，你拿到的东西通常比一次写完的 Prompt 要好。

```text
好的 Prompt 是迭代出来的。
你不需要一次写对。
你只需要每次看完回复后，指出"哪里不对"。
```

这就是为什么 [Agent 的工作方式](/docs/concepts/what-is-agent) 里强调"指令→规划→反馈→迭代"的循环——这跟写 Prompt 是同一个道理。

<h2 id="not-magic">"神级 Prompt"为什么会失效</h2>

你可能遇到过这种情况：网上抄了一个号称很牛的 Prompt，结果 AI 回复完全不搭。

原因是：**Prompt 的有效性取决于上下文。**

同样的 Prompt，AI 不同、背景信息不同、任务场景不同，结果完全不同。

一个帮别人写广告文案的 Prompt，你拿来找代码 bug，当然不灵。

一个英文 Prompt，你翻译成中文直接用，效果也可能不一样——因为模型的训练数据里中英文的分布不同。

```text
别人的 Prompt 可以参考思路，但不要直接照搬。
你比任何人更了解你的任务、你的读者、你的标准。
你的 Prompt 应该比任何模板都好用。
```

<h2 id="for-real-people">给不写代码的人：三个最常用场景</h2>

你不是程序员，但你大概率会遇到这些场景：

**场景一：写文档**

```text
帮我写一份会议纪要。
会议主题：Q2 销售策略调整
参会人：销售总监、三个区域经理
核心结论：华北区要增加人手，华南区降价策略效果不好要停
输出格式：分"背景-讨论-结论-下一步"四个部分，每部分不超过 200 字
不要用 AI 生成的套话，要像人写的。
```

**场景二：整理信息**

```text
下面是一段访谈录音转文字。帮我整理成三个部分：
1. 核心观点（不超过 5 条）
2. 值得跟进的问题（不超过 3 条）
3. 一句话总结
不要添加原文没有的信息。不要评价观点对不对。
[粘贴文字]
```

**场景三：做 PPT 内容**

```text
帮我为一份 PPT 写大纲。
题目：2025 上半年生产部门总结
受众：厂长和各部门主管
时长：15 分钟
要包含：产量数据、质量问题、设备维护、下半年计划
每页只写要点，不要完整句子。总共不超过 12 页。
不要用"赋能""抓手""闭环"这类词。
```

这三个例子有一个共同点：都不是"一句话指令"。每个都包含了任务、边界、验收标准、背景。

但你不需要背格式。你只需要每次问自己：

```text
AI 知道我要什么吗？
AI 知道什么算做完吗？
AI 知道什么不能做吗？
AI 有足够的背景信息吗？
```

<h2 id="remember">记住一件事</h2>

```text
Prompt 不是咒语，是工作指令。
你不需要学一个新语言。
你只需要把给同事派活的方式，
用文字写给 AI。
```
