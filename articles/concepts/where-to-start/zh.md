# 从哪开始

你读完了前面七章。现在你知道：

- AI 是一个读过很多书但记性不好的实习生
- 三个底层突破让它从"聊天玩具"变成了"生产工具"
- Agent 不是更聪明的聊天机器人，是能操作你电脑的"员工"
- Prompt 不是咒语，是工作指令——说清楚四件事就行
- 管好上下文比写好 Prompt 更重要——墙上、桌上、垃圾桶
- AI 自信不等于正确——你懂的领域你审，不懂的交叉验证
- 模型是大脑、工作台是身体、平台是后勤——选工作台比选模型重要

现在最后一个问题：**我第一步到底做什么？**

<h2 id="three-paths">三条路</h2>

不同的人走不同的路。先判断你是谁：

```text
A. 我想先感受一下，不确定要不要装东西
B. 我有具体任务想做，不想再自己手动干了
C. 我想系统性地把 AI 用起来，当日常工具
```

<h2 id="path-a">A 路：纯尝鲜</h2>

**你不需要装任何东西。**

打开任何一个 AI 聊天网页——ChatGPT、Claude、Kimi、豆包，随便哪个。把你在 [跟 AI 说话，不是念咒](/docs/concepts/what-is-prompt) 里学到的东西用一遍：

1. 想一个你工作里的小任务（写一封邮件、总结一段文字、翻译一个文档）
2. 写清楚：要什么、不要什么、什么算做完、有什么背景
3. 发过去，看结果
4. 改一句你的指令，再发一次，看结果有什么不同

做完这四步，你就已经比 90% 的人更懂怎么用 AI 了。

```text
尝鲜目标：体验一次"说清楚指令 → 得到对的结果"的感觉。
时间：10 分钟。
不需要装任何东西。
```

<h2 id="path-b">B 路：有任务，想干活</h2>

你有具体的任务想在 AI 的帮助下完成。比如整理报表、写文档、做 PPT、分析数据。

**你需要的是一站式流程：装工作台 → 装能力包 → 跑通一个任务。**

**第一步：选一个工作台**

```text
用 GPT 模型 → 装 Codex
用 Claude 模型 → 装 Claude Code
不确定 → 先装 Codex，当前最简单的新手入口
```

具体的安装步骤在 [Runtime / Codex](/docs/runtime/codex) 或 [Runtime / Claude Code](/docs/runtime/claude-code)。安装过程通常是一条命令，几分钟搞定。

**第二步：装一个 Skill**

Skill 是给 Agent 的能力包。你想做什么，就装对应的 Skill：

| 你想做什么 | 装哪个 Skill |
| --- | --- |
| 处理 Word / Excel / PPT / PDF | [办公文档](/docs/skills/office-docs) |
| 生成图片、海报、封面 | [SorryCode Image2](/docs/skills/sorrycode-image2) |
| 做一页纸、简历、作品集 | [Kami](/docs/skills/kami) |
| 做网页 PPT | [藏师傅的 PPT Skill](/docs/skills/magazine-web-ppt) |

**第三步：跑通一个任务**

不要一上来就做复杂的。选一个小任务，从头跑到尾。

比如："帮我把这份 Excel 报表里的数据按月份汇总，画三张趋势图，生成一份 PDF。"

复制这句话给你的 Agent，它会先读文件、给计划、等你确认、然后干活。

```text
B 路目标：装好工作台，跑通第一个任务。
时间：30-60 分钟（安装 + 第一个任务）。
关键：跳过计划审查 = 浪费时间和钱。
```

<h2 id="path-c">C 路：系统性用起来</h2>

你已经跑通了几个任务，想把 AI 变成日常工具。

**你需要的建立你的项目规则。**

创建一个文件夹，放你的工作文件。在文件夹里建一个 `AGENTS.md`（如果你用 Codex）或 `CLAUDE.md`（如果你用 Claude Code）。

这个文件里写：

```text
- 这个项目是什么
- 目录结构
- 常用命令和工具
- 代码/写作风格规则
- 哪些文件不要改
- 完成任务前要检查什么
```

然后每次让 Agent 干活之前，先让它读这个文件。

更多的管理方法，看 [Agent 进阶](/docs/agent-infra/overview)。

```text
C 路目标：建立你的第一个项目规则文件。
时间：20 分钟写完第一版，以后逐步更新。
关键：只写长期有效的规则，不要把临时任务写进去。
```

<h2 id="dont">三件不要做的事</h2>

不管你选哪条路，记住三个"不要"：

```text
1. 不要追模型名字。
   先追任务。把一件事跑通，比知道十个模型名字重要。

2. 不要同时试太多工具。
   选一个工作台，装好，跑通一个任务。再考虑换。

3. 不要一次学太多。
   前面七章的内容，你不是今天就要全部掌握。
   用到哪，学到哪。路由先于加载。
```

<h2 id="recap">这本书讲了什么</h2>

```text
怎么把一个"读过全世界所有书但记性不好的实习生"，
训练成能替你干活的"员工"。
```

路径是：

```text

→ 选择它（7 章）

```
