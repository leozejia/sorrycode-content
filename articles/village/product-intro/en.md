# Create a Product Intro

<h2 id="goal">Goal</h2>

Turn a product, service, or small project into a one-page intro you can send to someone else.

This is a `one-pager`: one page that explains one thing clearly.

<h2 id="prepare">What You Need</h2>

- [Kami](/docs/skills/kami) installed
- product or project name
- your raw description

<h2 id="steps">Steps</h2>

1. Fill in the product information below
2. Ask `Kami` for the page structure first
3. Confirm the structure before generating the final artifact
4. Ask it to check whether the reader knows what to do next

<h2 id="inputs">Fill This In</h2>

```text
Product name:
Target user:
Problem it solves:
Top 3 selling points:
Price, entry point, or usage path:
What the reader should do next:
Tone: restrained / energetic / professional / creator-style
```

<h2 id="prompt">Copy This</h2>

```text
Turn the information below into a one-pager product intro for someone seeing it for the first time. Give me the page structure first, then wait for my confirmation before generating the final artifact.
```

<h2 id="done">Completion Check</h2>

The product intro should make clear:

- what the product is
- what problem it solves
- why the benefits matter to the target user
- what the reader should do next

<h2 id="stuck">If You Get Stuck</h2>

If it sounds too much like an ad, say: `Use fewer slogans and more concrete user scenarios.`

If it feels scattered, say: `Compress this into 4 sections, with only one key point per section.`
