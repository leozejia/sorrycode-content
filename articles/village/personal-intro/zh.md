# 做一份个人介绍

<h2 id="goal">任务目标</h2>

做一页能发给第一次认识你的人的个人介绍。

它不是完整简历，而是更轻的一页材料：讲清你是谁、你做什么、你能帮别人什么。

<h2 id="prepare">准备物品</h2>

- 已安装 [Kami](/docs/skills/kami)
- 你的基本介绍
- 你希望别人记住你的一个关键词

<h2 id="steps">通关步骤</h2>

1. 先按模板写出原始信息
2. 让 `Kami` 先给页面结构
3. 选择语气：专业、亲和、创作者风格，或更正式
4. 确认结构后生成成品文档
5. 最后让它帮你删掉空话

<h2 id="inputs">填写这些信息</h2>

```text
我是谁：
我现在做什么：
我擅长什么：
代表作品、经历或项目：
别人可以找我做什么：
我希望别人记住我的一句话：
语气：专业 / 亲和 / 创作者风格 / 正式
```

<h2 id="prompt">复制这句话</h2>

```text
请把下面这些信息整理成一份 One-pager 个人介绍，适合发给第一次认识我的人。先给我页面结构，等我确认后再生成成品文档。
```

<h2 id="done">通关标志</h2>

这份个人介绍应该做到：

- 一页能说清你是谁
- 能说清你能帮别人什么
- 有可信的经历或作品支撑
- 读者知道怎么继续联系或合作

<h2 id="stuck">卡关怎么办</h2>

如果它写得太像简历，让它改成：`不要按简历写，请按第一次介绍给合作对象的语气写。`

如果它写得太虚，让它追问你：`请先问我 5 个问题，帮我补足具体经历和代表作品。`
